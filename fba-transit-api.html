Subscribe Callback

订阅回调 Doc ID: 111697

Page name: Subscription Callback

Function Description

The subscription notification system is a widely used system that can be used to send various types of notifications to system integrators, such as inventory changes, order labeling, etc. It can effectively reduce invalid polling data requests, save server resources, and greatly improve data timeliness.

● To enable the callback function, you must first subscribe to messages. For detailed information, please refer to:

● Timely notification of data changes to the client, improving data timeliness;

● Change the client’s calling mode from client polling to subscription notification pulling, improving performance and reducing server resource consumption;

Note:

1. This interface is the callback address interface set when WMS business data changes actively call subscription notifications, notifying the subscriber to obtain data updates;

2. Retry callback up to 5 times, once per minute;

3. After the subscriber receives the callback notification, it should obtain data according to the defined interface;

Status Description:

● Return Pre-advice Order Status Description: 1: Draft, 2: Submitted, 3: Reviewed, 4: Processing, 5: Processed, 6: Voided, 7: Rejected

● Return Order Status Description: 1: Temporary Storage, 2: Pending Counting, 3: Processing, 4: Completed

● Return Processing Order Status Description: 1: Draft, 2: Reviewed, 3: Processing, 4: Processed, 5: Voided

● Boxing Task Order Status Description: 1: Draft, 2: Reviewed, 3: Boxing, 4: Boxed, 5: Voided

● Outbound Order Status Description: 1: Draft, 2: Pending Shipment, 3: Picked, 4: Palletized, (Picked, Palletized -> Shipping), 5: Shipped, 6: Voided, 7: Exception, 9: Delivered

● Transfer Order Status Description: 0: Draft, 1: Pending Receipt, 2: Receiving, 3: Received, 4: Shipped, 5: Delivered, 6: Exception, 7: Voided
